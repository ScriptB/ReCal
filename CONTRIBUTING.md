# Contributing to ReCal

Thanks for your interest in contributing. This guide covers setup, structure, and how to submit changes.

## Prerequisites

- Visual Studio 2022 or later
- .NET Framework 4.8
- Git

## Setup

1. Fork the repository.
2. Clone your fork locally.
3. Open `ReCal.sln` in Visual Studio.
4. Build the project to confirm the environment is working.

## Project Structure

- `src/` WPF application source
- `assets/` icons and images
- `docs/` documentation
- `tools/` build and release scripts
- `ReCal.sln` solution file

## Code Style

- Follow C# naming conventions.
- Keep methods small and focused.
- Use async/await for I/O operations.
- Add XML comments for public types and members where helpful.

## Submitting Changes

1. Create a new branch for your work.
2. Make your changes and test them.
3. Commit with clear messages.
4. Push to your fork.
5. Open a pull request.

## Pull Request Guidelines

- Provide a clear description of changes.
- Link related issues if any.
- Include screenshots for UI changes.
